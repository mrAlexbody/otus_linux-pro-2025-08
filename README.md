
# Курс «Administrator Linux. Professional». 

---

## Анализ репозитория

Репозиторий содержит 29 тематических директорий (HW01–HW29) и одну проектную (HW30).  
Каждая директория, как правило, включает:
- `README.md` — описание задания, цели, требования;
- скрипты (`bash`, `python`) и конфигурационные файлы (`jinja`, `dockerfile`, `ansible` и т.д.);
- примеры вывода, проверочные команды.

Основные технологии: **Linux (ядро, загрузка, systemd), системы хранения (mdadm, LVM, ZFS, NFS), управление пакетами, Bash, процессы, SELinux, Ansible, Vagrant, Docker, Prometheus, PAM, логирование, бэкапы, сеть (DHCP, iptables, OSPF, VPN, VLAN/LACP), LDAP, DNS, web-серверы, MySQL, PostgreSQL**. Задания имеют возрастающую сложность и строятся на виртуальных машинах (Vagrant, VirtualBox).

## Задачник по курсу «Administrator Linux. Professional»

### 1. Ядро и загрузка системы

#### **HW01: Обновление ядра системы**
- **Цель**: научиться обновлять ядро в CentOS/RHEL без использования стандартных репозиториев.
- **Задачи**:
  1. Установить ядро из исходников или из стороннего репозитория (ELRepo).
  2. Настроить загрузчик (GRUB2) для выбора нового ядра по умолчанию.
  3. Проверить, что система загружается с новым ядром.

#### **HW07: Загрузка системы (Bootloader)**
- **Цель**: работа с GRUB2, восстановление загрузчика, добавление параметров ядра.
- **Задачи**:
  1. Сломать и восстановить загрузчик GRUB2.
  2. Добавить параметр ядра (например, `single` или `rd.break`) для аварийного входа.
  3. Настроить таймаут и порядок загрузки.

#### **HW08: Инициализация системы (Systemd)**
- **Цель**: управление сервисами systemd, создание собственных unit-файлов.
- **Задачи**:
  1. Написать unit-файл для пользовательского скрипта.
  2. Добавить зависимость от сети и таймер для периодического запуска.
  3. Проанализировать журнал systemd.

### 2. Управление хранилищами и файловыми системами

#### **HW02: Работа с mdadm**
- **Цель**: создание программных RAID-массивов.
- **Задачи**:
  1. Создать RAID 1 из двух дисков.
  2. Сымитировать отказ одного диска и выполнить замену.
  3. Настроить мониторинг состояния массива.

#### **HW03: Работа с LVM**
- **Цель**: гибкое управление томами.
- **Задачи**:
  1. Создать VG, LV, файловую систему.
  2. Расширить LV без остановки работы.
  3. Уменьшить LV (с предварительным уменьшением ФС).
  4. Сделать снапшот и восстановить данные.

#### **HW04: Практические навыки работы с ZFS**
- **Цель**: изучить ZFS (пулы, файловые системы, снапшоты).
- **Задачи**:
  1. Создать пул ZFS с разными типами vdev (mirror, raidz).
  2. Включить сжатие и дедупликацию.
  3. Настроить снапшоты и репликацию.

#### **HW05: Работа с NFS**
- **Цель**: развернуть NFS-сервер и настроить экспорт.
- **Задачи**:
  1. Установить и настроить NFSv4.
  2. Экспортировать каталог с опциями `rw, sync, no_root_squash`.
  3. Подключить на клиенте и проверить права доступа.

---

### 3. Системное администрирование

#### **HW06: Управление пакетами**
- **Цель**: работа с RPM/YUM/DNF, сборка собственного RPM-пакета.
- **Задачи**:
  1. Найти, установить, удалить пакет.
  2. Собрать RPM из исходников (или написать простой spec-файл).
  3. Настроить локальный репозиторий.

#### **HW09: BASH**
- **Цель**: написание скриптов для автоматизации.
- **Задачи**:
  1. Скрипт для бэкапа каталога с ротацией.
  2. Скрипт для сбора информации о системе (CPU, RAM, диски).
  3. Использование `getopts`, `trap`, проверка ошибок.

#### **HW10: Управление процессами (LSOF)**
- **Цель**: диагностика открытых файлов, портов, процессов.
- **Задачи**:
  1. Найти процесс, занимающий порт 80.
  2. Определить, какие файлы использует процесс.
  3. Убить процесс по имени или по PID.

### 4. Безопасность и контроль доступа

#### **HW11: SELinux**
- **Цель**: настройка SELinux в целевых режимах.
- **Задачи**:
  1. Перевести SELinux в режим enforcing.
  2. Создать новый контекст для нестандартного порта веб-сервера.
  3. Написать локальный модуль политики.

#### **HW16: PAM**
- **Цель**: настройка аутентификации через PAM.
- **Задачи**:
  1. Ограничить доступ по SSH для определённой группы.
  2. Настроить двухфакторную аутентификацию (Google Authenticator).
  3. Вести учёт неудачных попыток входа.

#### **HW25: LDAP**
- **Цель**: централизованная аутентификация через OpenLDAP.
- **Задачи**:
  1. Установить и настроить OpenLDAP-сервер.
  2. Добавить пользователей и группы.
  3. Настроить клиент для аутентификации через LDAP.

---

### 5. Автоматизация и контейнеризация

#### **HW12: Ansible**
- **Цель**: автоматизация конфигурации с помощью Ansible.
- **Задачи**:
  1. Написать playbook для установки и настройки веб-сервера.
  2. Использовать переменные, хендлеры, шаблоны Jinja2.
  3. Применить роли (roles).

#### **HW13: Vagrant**
- **Цель**: создание и управление виртуальными средами.
- **Задачи**:
  1. Написать Vagrantfile для развёртывания двух машин.
  2. Настроить сеть между ними (private network).
  3. Использовать provision (shell, ansible).

#### **HW14: Docker**
- **Цель**: контейнеризация приложений.
- **Задачи**:
  1. Написать Dockerfile для простого веб-приложения.
  2. Собрать образ и запустить контейнер.
  3. Настроить volume для сохранения данных.
  4. Организовать связь между контейнерами (docker-compose).

---

### 6. Мониторинг и логирование

#### **HW15: Prometheus**
- **Цель**: сбор метрик и мониторинг.
- **Задачи**:
  1. Развернуть Prometheus и Node Exporter.
  2. Настроить сбор метрик с нескольких хостов.
  3. Создать простые запросы (PromQL) и настроить алертинг.

#### **HW17: LOG**
- **Цель**: централизованный сбор и анализ логов.
- **Задачи**:
  1. Настроить syslog (rsyslog) для отправки логов на центральный сервер.
  2. Реализовать ротацию логов (logrotate) с пользовательскими параметрами.
  3. Проанализировать логи с помощью `journalctl`.

---

### 7. Резервное копирование

#### **HW18: Backup**
- **Цель**: реализация стратегии бэкапов.
- **Задачи**:
  1. Создать скрипт для инкрементного бэкапа с использованием rsync.
  2. Настроить автоматическое выполнение через cron.
  3. Восстановить данные из бэкапа.

### 8. Сетевые технологии

#### **HW19: Network**
- **Цель**: базовая настройка сетевых интерфейсов, маршрутизация.
- **Задачи**:
  1. Настроить статические IP, добавить маршруты.
  2. Организовать NAT между двумя подсетями.
  3. Проверить работу с помощью `tcpdump`.

#### **HW20: DHCP**
- **Цель**: развёртывание DHCP-сервера.
- **Задачи**:
  1. Установить и настроить ISC DHCP-сервер.
  2. Создать статические привязки (по MAC).
  3. Настроить опции (маршрут, DNS).

#### **HW21: Iptables**
- **Цель**: настройка межсетевого экрана.
- **Задачи**:
  1. Написать правила для фильтрации трафика.
  2. Настроить NAT для доступа в интернет.
  3. Сохранить правила и сделать их постоянными.

#### **HW22: OSPF**
- **Цель**: динамическая маршрутизация с OSPF (FRR или Quagga).
- **Задачи**:
  1. Настроить OSPF на двух маршрутизаторах.
  2. Проверить обмен маршрутами.
  3. Настроить аутентификацию OSPF.

#### **HW23: VPN**
- **Цель**: организация защищённого канала (OpenVPN или WireGuard).
- **Задачи**:
  1. Развернуть VPN-сервер.
  2. Сгенерировать ключи и настроить клиент.
  3. Проверить маршрутизацию трафика через VPN.

#### **HW24: VLAN and LACP**
- **Цель**: настройка VLAN и агрегации каналов (bonding).
- **Задачи**:
  1. Создать VLAN-интерфейсы на виртуальных машинах.
  2. Настроить bonding (mode 4 — 802.3ad) с двумя интерфейсами.
  3. Проверить отказоустойчивость.

### 9. Сетевые сервисы

#### **HW26: DNS**
- **Цель**: работа с DNS (BIND).
- **Задачи**:
  1. Настроить кэширующий DNS-сервер.
  2. Создать мастер-зону для внутреннего домена.
  3. Настроить slave-сервер.

#### **HW27: WEB**
- **Цель**: развёртывание веб-сервера (nginx/apache) с балансировкой.
- **Задачи**:
  1. Установить nginx, настроить виртуальные хосты.
  2. Настроить балансировку нагрузки на два бэкенда.
  3. Включить HTTPS (самоподписанный сертификат).

#### **HW28: MySQL**
- **Цель**: администрирование MySQL/MariaDB.
- **Задачи**:
  1. Установить, создать базу и пользователя.
  2. Настроить репликацию master-slave.
  3. Выполнить резервное копирование и восстановление (mysqldump, xtrabackup).

#### **HW29: PostgreSQL**
- **Цель**: администрирование PostgreSQL.
- **Задачи**:
  1. Установить, настроить параметры (shared_buffers, wal).
  2. Создать базу, таблицы, пользователей.
  3. Настроить потоковую репликацию.
